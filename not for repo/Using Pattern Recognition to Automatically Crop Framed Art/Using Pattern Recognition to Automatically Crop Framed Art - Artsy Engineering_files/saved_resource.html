            Right you are @P! You'll notice that the output of the code linked in the post includes a csv file with the coordinates of the cropping shape, the four corners of a rotated rectangle ([line](https://gist.github.com/ily....

            From these 4 coords, its straightforward to calculate skew, and rotate the image. We also know the exact width and center of the image from the these coords, and use that to crop the now rotated image. Note that in our case, we have also done the edge detection on a smaller version of the image (for performance reasons) and also need to blow up our cropping width and center to the 10-15x larger original image.
